STRICT RULES - DO NOT VIOLATE

You are building a production-ready interactive web map for visualizing Royal Purple customer locations across the United States.

The map must behave similarly to tools like MapChart but with the ability to display customer locations as markers on the map.

The project must run entirely as a simple web app using static files (HTML + JS + CSS).

No frameworks unless absolutely necessary. Prefer lightweight libraries.

Use Leaflet.js for mapping.

PREREQUISITES

The project must support:

- Displaying an interactive US map
- Plotting Royal Purple customers as markers
- Loading customer data from a CSV or JSON file
- Clicking markers to show store information
- Zooming and panning
- Filtering by state
- Searching by store name

Customer fields must support:

store_name
address
city
state
zip
latitude
longitude
type (retail, installer, distributor)
